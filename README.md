# Exercice Guitare

A French-language guitar practice PWA built with React, TypeScript, and Vite. Supports note recognition and chord memorization with spaced-repetition weighting and optional pitch detection via microphone.

## Source layout

```
src/
  App.tsx               # Root component — view routing, settings/stats persistence
  main.tsx              # React entrypoint
  index.css             # Global styles and CSS variables
  hooks/
    usePitchDetection.ts  # WebAudio microphone → pitch detection hook
    useSession.ts         # Practice session lifecycle hook
  lib/
    constants.ts          # Notes, chords, chord presets/progressions, voicings
    pitch.ts              # Frequency → note conversion
    stats.ts              # Stats and weights persistence (localStorage)
    summarizeSession.ts   # Aggregates raw session results into summary
    util.ts               # Weighted random, TTS, formatting helpers
  components/
    shared.module.css     # Shared CSS classes (typography, layout)
    ui/                   # Reusable atoms/molecules
      ChordDiagram/       # SVG chord fingering diagram
      Icon/               # SVG sprite icon wrapper
      NotesPicker/        # Note enable/disable picker
      ProgressDot/        # Mastery level indicator dot
      Toggle/             # Toggle switch
    views/                # Full-page views rendered by App.tsx
      ConfigView/         # Exercise configuration (interval, notes, chords)
      DebugView/          # Pitch detection debug panel
      ProgressView/       # Progress dashboard with mastery levels
      SessionView/        # Active practice session
      SummaryView/        # Post-session statistics
      WelcomeView/        # Welcome screen with mode selection
```

Each component folder contains `index.tsx` and `index.module.css`.

## Setup

```bash
npm install                       # also runs `prepare` → installs the husky git hooks
npx playwright install chromium   # browser used by the E2E tests
```

The pre-commit hook (`.husky/pre-commit`) runs `npm run typecheck` and the no-mic
BDD suite (`npm run test:dev`) before each commit.

## Development

```bash
npm run dev       # dev server at http://localhost:5173
npm run build     # production build
npm run deploy    # build + deploy to GitHub Pages
```

## Testing

End-to-end behavior tests are written in Gherkin (Cucumber) and driven through a
real browser with Playwright — see `tests/`. They exercise the actual UI and real
`localStorage` with no application-code mocking. The only faked input is the
microphone, used by a single tagged smoke test.

```bash
npm test          # build, serve the production bundle, run the no-mic suite headless
npm run test:dev  # same suite against the Vite dev server (faster local loop)
npm run test:headed   # run with a visible browser for debugging
npm run test:audio    # the @audio smoke test: real pitch-detection pipeline fed a
                      # fake 440 Hz (La) WAV via Chromium fake-audio capture
```

- Feature files: `tests/features/*.feature`. The full Welcome → Chords config →
  Session → Summary → Welcome journey lives in `full-journey.feature`.
- Step definitions: `tests/steps/practice.steps.ts`; browser/world setup in
  `tests/support/`.
- To add coverage, write a new scenario in a `.feature` file and reuse the existing
  steps (or add one). Prefer the `data-testid` hooks already present in the views.
- The fake-audio tone is generated by `npm run tones` (`tests/fixtures/gen-tones.mjs`).
  The `@audio` test requires the full Chromium build (`channel: "chromium"`) and is
  Chromium-only — keep it out of the default run if it proves flaky in CI.

## Scripts

- `scripts/simulate-sr.mjs` — simulate spaced repetition weight progression
- `scripts/wait-for-deploy.js` — poll GitHub Pages until a deploy is live
